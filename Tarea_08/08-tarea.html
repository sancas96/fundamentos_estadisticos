fun_zero <- function(x){
  if(x==0){
    return(.01)
  }
  else{
    return(x)
  }
}

Los clculos de un anlisis de componentes principales involucran la matriz de covarianzas emprica \(G\) (estimaciones plug-in)

\[G_{jk} = \frac{1}{88}\sum_{i=1}^88(x_{ij}-\bar{x_j})(x_{ik}-\bar{x_k})\]

para \(j,k=1,2,3,4,5\), y donde \(\bar{x_j} = \sum_{i=1}^88 x_{ij} / 88\) (la media de la i-sima columna).

G <- cov(marks) * 87 / 88

  1. Proponemos el siguiente modelo simple para puntajes correlacionados:

\[\textbf{x}_i = Q_i \textbf{v}\]

donde \(\textbf{x}_i\) es la tupla de calificaciones del i-simo estudiante, \(Q_i\) es un nmero que representa la habilidad del estudiante y \(\textbf{v}\) es un vector fijo con 5 nmeros que aplica a todos los estudiantes. Si este modelo simple fuera cierto, entonces nicamente el \(\hat{\lambda}_1\) sera positivo y \(\textbf{v} = \hat{v}_1\). Sea \[\hat{\theta}=\sum_{i=1}^5\hat{\lambda}_i\] el modelo propuesto es equivalente a \(\hat{\theta}=1\), inculso si el modelo es correcto, no esperamos que \(\hat{\theta}\) sea exactamente uno pues hay ruido en los datos.

theta_hat <- lambda[1]/sum(lambda)
theta_hat

El valor de \(\hat{\theta}\) mide el porcentaje de la varianza explicada en la primer componente principal, qu tan preciso es \(\hat{\theta}\)? La complejidad matemtica en el clculo de \(\hat{\theta}\) es irrelevante siempre y cuando podamos calcular \(\hat{\theta}^*\) para una muestra bootstrap, en esta caso una muestra bootsrtap es una base de datos de 88 \(\times\) 5 \(\textbf{X}^*\), donde las filas \(\bf{x_i}^*\) de \(\textbf{X}^*\) son una muestra aleatoria de tamao 88 de la verdadera matriz de datos.
